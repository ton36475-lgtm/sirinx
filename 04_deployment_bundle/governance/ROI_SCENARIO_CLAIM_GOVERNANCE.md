# ROI Scenario Claim Governance

## Purpose

Allow SIRINX to present executive-grade ROI stories without turning model outputs into guaranteed financial claims.

Scenario calculations must follow `governance/ROI_SCENARIO_MODELING.md`. This document governs how model outputs may be turned into claims.

## Claim Status Taxonomy

| Status | Meaning | Public Use | Approval Required |
| --- | --- | --- | --- |
| `Verified` | Backed by actual bills, measured generation, signed proposal values, or post-install data | Yes | Evidence review |
| `Estimate` | Calculator result using user-provided bill, tariff, system size, or irradiance assumptions | Yes with assumptions | Sales/engineering review |
| `Validated Sales Scenario` | Sales or boardroom model supported by field context, prior proposal logic, or reviewed assumptions | Yes with assumptions and review marker | Human review before publish |
| `Target Model` | Target-state planning model for savings, payback, LCOE, or carbon impact | Yes with assumptions and sensitivity range | Human review before publish |
| `Case-Study Hypothesis` | Case-study style narrative derived from field patterns, awaiting customer/project verification | Limited, with hypothesis label | Human review before publish |
| `Review Required` | Marketing hook, legal/tax/ESG statement, or high-impact claim awaiting validation | Draft only | Owner approval |

## High-Impact Example Rule

Examples such as electricity cost `250,000 -> 50,000` must never be written as guaranteed claims.

If the example is based on a pilot project, reference `governance/PILOT_PROJECT_STATUS.md`. Pilot status can strengthen a validated sales scenario or case-study hypothesis, but it does not make the outcome guaranteed.

If the example is generated by a model, reference the ROI model record from `governance/ROI_SCENARIO_MODELING.md`.

Preserved target model:

- `governance/records/ROI_TARGET_MODEL_250K_TO_50K_70K_2026-04-23.md`: `250,000 -> 50,000-70,000` target business case scenario. Claim status: `Target Model`, review required, not verified and not guaranteed.

Allowed descriptions for the `250,000 -> 50,000-70,000` range:

- target scenario
- ROI model input
- case-study objective

Do not describe the range as achieved savings, delivered result, accepted handover, verified track record, guaranteed outcome, or guaranteed universal outcome.

Allowed labels:

- `Validated Sales Scenario`
- `Target Model`
- `Case-Study Hypothesis`
- `Review Required`

Not allowed unless backed by actual before/after bills and measured post-install data:

- `Verified`
- "guaranteed"
- "guaranteed universal outcome"
- "แน่นอน"
- "ทุกโรงงาน"
- "ลดได้ 80% ทุกเคส"

## Required Wording

Use language such as:

- "validated sales scenario"
- "target model"
- "case-study hypothesis"
- "target scenario"
- "ROI model input"
- "case-study objective"
- "ภายใต้สมมติฐาน"
- "ต้องตรวจสอบด้วยข้อมูลหน้างานจริง"
- "ไม่ใช่การรับประกันผลลัพธ์"

## Rollback Note

If a public asset accidentally presents a modeled ROI as a guaranteed result, remove or replace that copy immediately, publish a corrected version with claim status and assumptions, and preserve the before/after diff.

## Audit Trail Note

Save:

- ROI model record and version
- claim copy before and after review
- assumption sheet or calculation input
- evidence links
- reviewer or approver
- validation output
- rollback target

## Current Decision

`250,000 -> 50,000` is allowed only as a validated sales scenario, target model, case-study hypothesis, or review-required draft. It is not a guaranteed claim.
