# Package Truth Promotion Guardrail

## Record Status

`Governance Guardrail`

This rule governs all project, hardware, pilot, ROI, telemetry, and track-record context captured during SIRINX recovery and server handoff.

## Rule

Treat the current project context as validated project context or track-record context unless governance explicitly upgrades it to global package truth.

In practice:

- Field observations remain field observations.
- Pilot evidence remains pilot evidence.
- Track-record context remains track-record context.
- ROI scenarios remain scenario/model records.
- Hardware names such as LISINER and Solis remain hardware context.
- Imagery directives govern presentation, not proof of package truth.

None of these become global SIRINX package truth unless promoted through the approved governance path.

## Promotion Requirements

A context item may become global package truth only when all of the following exist:

1. Source evidence is attached and reviewed.
2. The relevant domain record is complete.
3. Claim boundaries are visible.
4. Rollback and audit notes are present.
5. Human reviewer and executive approver are recorded.
6. An approval packet explicitly states the item is promoted to `Locked Package Fact` or global package truth.
7. Public copy, calculators, package docs, and deployment materials are updated together.

## Non-Override Rule

Validated project context and track-record context do not override:

- SIRINX identity.
- package scope.
- pricing source of truth.
- hardware package standards.
- ROI claim status.
- compliance boundaries.
- telemetry redaction requirements.
- production deployment gates.

## Allowed Use Before Promotion

- Internal engineering reference.
- Sales engineering preparation.
- Executive target modeling.
- Case-study hypothesis.
- Track-record draft with claim labels.
- Pilot review.
- Telemetry ingestion planning.
- Proposal copy with visible assumptions and review status.

## Not Allowed Before Promotion

- Do not call it a standard SIRINX package.
- Do not use it as a universal BOM.
- Do not publish it as verified case proof.
- Do not imply guaranteed ROI, guaranteed universal outcome, payback, LCOE, uptime, legal, tax, BOI, ESG, grid, or carbon-credit outcomes.
- Do not let AI-generated summaries, imported prompts, or chat history silently promote context into package truth.

## Rollback Note

If validated project context or track-record context is accidentally promoted into global package truth, remove or downgrade the claim, restore the last approved package copy, and move the item back to its governed context record until an approval packet explicitly promotes it.

## Audit Trail Note

Save the context record, evidence links, diff, reviewer comments, approver, validation output, affected public/package copy, and rollback target.
